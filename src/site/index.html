Support

Browser-based PowerShell and shared Windows terminals from one host.

Browser-based PowerShell

TermiWeb prefers PowerShell 7 when available and falls back to Windows PowerShell when needed, so the browser surface stays useful on real Windows hosts.

Shared browser instances

Open shared terminal instances through the browser UI and attach to the same live shell from more than one device.

Packaged Windows run surface

Setup, start, restart, stop, uninstall, and optional before-sign-in auto-start are part of the packaged run surface.

Authenticated browser sessions

Use one configured app password for local and LAN access, with browser sessions that survive normal server restarts until logout or expiry.

Shared width per instance

Each instance keeps one shared column width across attached devices, with 80 columns as the default starting point for consistent terminal sizing when switching devices.

Phone-friendly controls

Mobile controls, selection mode, copy handling, and the fallback paste field are all built into the browser UI.

First Run

Start from the package root and let the setup path do the heavy lifting.

  1. Unpack the release into a user-writable folder.
  2. Open 1.Start-Here.md and read DISCLAIMER.md.
  3. Run Set Up TermiWeb.cmd for the guided first-run path.
  4. Approve the Windows elevation prompt when TermiWeb starts.
  5. Open http://127.0.0.1:22443 if setup does not open it for you.
  6. From another device on the same LAN, browse to the host device’s LAN URL to attach to the same live shell.

Trust

Windows hosts only

Trusted-network-first

WAN is your responsibility

Single-user auth

Includes single-user auth without TLS to keep it simple.
