What 0.1 Supports

One release, one host, shared across browsers.

Shared browser instances

Open shared terminal instances through the browser UI and attach to the same live shell from more than one device.

Packaged Windows run surface

Setup, start, restart, stop, uninstall, and optional before-sign-in auto-start are part of the intended 0.1 path.

Authenticated browser sessions

Use one configured app password for local and LAN access, with browser sessions that survive normal server restarts until logout or expiry.

Phone-friendly controls

Mobile controls, selection mode, copy handling, and the fallback paste field are all part of the shipped UI.

Shared width per instance

Each instance keeps one shared column width across attached devices, with 80 columns as the default starting point.

PowerShell-first shell path

TermiWeb prefers PowerShell 7 when available and falls back to Windows PowerShell when needed.

First Run

Start from the package root and let the setup path do the heavy lifting.

  1. Unpack the release into a user-writable folder.
  2. Open 1.Start-Here.md and read DISCLAIMER.md.
  3. Run Set Up TermiWeb.cmd for the guided first-run path.
  4. Approve the Windows elevation prompt when TermiWeb starts.
  5. Open http://127.0.0.1:22443 if setup does not open it for you.
  6. From another device on the same LAN, browse to the host device’s LAN URL to attach to the same live shell.

Trust Boundaries

0.1 is honest about what it is and what it leaves to you.

Windows hosts only

0.1 targets Windows hosts and uses an elevated-only shell path.

Trusted-network-first

The intended default is a trusted Windows machine on a private network.

WAN is your responsibility

Internet or cross-site exposure is possible, but securing it is up to you. 0.1 does not claim to be a full remote-access stack.

No built-in multi-user stack

0.1 does not currently ship multi-user auth, built-in TLS termination, or a turnkey public-exposure workflow.
