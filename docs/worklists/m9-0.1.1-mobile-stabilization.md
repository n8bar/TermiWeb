# M9 0.1.1 Mobile Stabilization

Milestone Timing: Mirrors `M9` in [Project plan](../PLAN.md): start `2026-04-27`, target end `2026-05-10`.

Status: Completed on 2026-05-01.

## Desired Outcome

TermiWeb should leave `M9` with a credible `0.1.1` patch release: the must-have mobile fixes from Findings `13` and `14` shipped, the release surface refreshed, and a clear local verification discipline that fits the current one-maintainer workflow.

## 1. Work Breakdown

1. [x] Define the `0.1.1` local release discipline.
   1. [x] Decide which local checks must pass for a `0.1.1` release candidate.
      Decision: `npm run typecheck`, `npm test`, `npm run build`, `npm run build:site`, and `npm run package:release` must pass locally before the patch release can ship.
   2. [x] Decide which task-specific manual mobile checks are required before the patch release can ship.
      Decision: manual checks must cover portrait-phone collapsed controls, expanded active-instance controls, columns changes, instance close, `Type`, modifier keys, arrows, `Home`, `End`, paste, select, copy, punctuation-heavy command entry, phone rotation, and workstation create/select/resize/close regression coverage.
   3. [x] Document the resulting `0.1.1` verification expectations where they belong.
      Decision: detailed verification expectations live in [the `0.1.1` mobile stabilization spec](../specs/v0.1.1-mobile-stabilization.md).

2. [x] Fix Finding 13 for `0.1.1`.
   1. [x] Define the collapsed mobile instance-control behavior that replaces the current too-small controls.
      Definition: collapsed mode should show one large active-instance touch target; width/columns controls and close should be available only after expanding or opening the already active instance controls as physically larger targets instead of cramped rail-sized buttons; the opened controls should align with the active card top and avoid forcing focus or opening the mobile keyboard; inactive collapsed sessions should only select on tap/click, while tapping/clicking the already active collapsed session toggles controls; `Type` remains available; close remains separate from selection once expanded; collapsed-control state remains browser-local.
   2. [x] Implement the chosen mobile layout and interaction changes.
   3. [x] Verify the fix against the local release checks plus real mobile checks.
      Verification: real-device mobile testing confirmed the active-session flyout, columns popover, preset application, inactive-session selection, and post-resize terminal rendering now behave without the earlier tap, keyboard, or stale-layout failures.

3. [x] Fix Finding 14 for `0.1.1`.
   1. [x] Identify the browser and input hints available for suppressing mobile text assistance in terminal entry.
      Decision: apply the strongest practical browser hints per input surface, including `autocapitalize="off"`, `autocomplete="off"`, `autocorrect="off"`, `spellcheck="false"`, and useful `enterkeyhint`/`inputmode` choices only when they do not restrict normal shell input. Do not use password-style or credential-shaped terminal capture as a workaround; password-manager support belongs on the actual TermiWeb login, not PTY input. If normal text capture still lets mobile prediction corrupt terminal entry, use a non-credential mobile capture type such as `url`.
   2. [x] Implement the best supported mitigations without breaking normal terminal input behavior.
   3. [x] Verify the fix against the local release checks plus real mobile checks.
      Verification: real-device mobile testing confirmed the non-credential capture fallback stops the predictive text corruption without reintroducing password-manager terminal overlays.

4. [x] Refresh the release surface and ship the `0.1.1` patch release.
   1. [x] Update the release notes, package metadata, artifact references, and any versioned site or README references needed for `0.1.1`.
      Updated `package.json`, `package-lock.json`, the website download surface, README release wording, and the [0.1.1 release publication draft](../v0.1.1-release-publication-draft.md).
   2. [x] Update any package-root docs, screenshots, or public copy that stop being honest once the `0.1.1` fixes land.
      README and website copy now mention the `0.1.1` mobile stabilization release. Existing screenshots remain representative of the current UI and do not need replacement for this patch.
   3. [x] Run the final local verification pass for the patch release.
      Verification passed: `npm run typecheck`, `npm test`, `npm run build`, `npm run build:site`, and `npm run package:release`. The release package was assembled at `artifacts/release/TermiWeb-0.1.1-windows-x64.zip`.
   4. [x] Publish `0.1.1` and close the milestone cleanly.
      Published GitHub release: `https://github.com/n8bar/TermiWeb/releases/tag/v0.1.1`. The `TermiWeb-0.1.1-windows-x64.zip` asset is attached, the GitHub Pages deploy for `main` passed, and the live site/download page now advertises the `0.1.1` package.
