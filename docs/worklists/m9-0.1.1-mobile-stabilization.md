# M9 0.1.1 Mobile Stabilization

Milestone Timing: Mirrors `M9` in [Project plan](../PLAN.md): start `2026-04-27`, target end `2026-05-10`.

This is the first `M9` worklist draft. It should be reviewed before execution is treated as approved direction.

## Desired Outcome

TermiWeb should leave `M9` with a credible `0.1.1` patch release: the must-have mobile fixes from Findings `13` and `14` shipped, the release surface refreshed, and a clear local verification discipline that fits the current one-maintainer workflow.

## 1. Work Breakdown

1. [x] Define the `0.1.1` local release discipline.
   1. [x] Decide which local checks must pass for a `0.1.1` release candidate.
   2. [x] Decide which task-specific manual mobile checks are required before the patch release can ship.
   3. [x] Document the resulting `0.1.1` verification expectations where they belong.

2. [ ] Fix Finding 13 for `0.1.1`.
   1. [x] Define the collapsed mobile instance-control behavior that replaces the current too-small controls.
   2. [ ] Implement the chosen mobile layout and interaction changes.
   3. [ ] Verify the fix against the local release checks plus real mobile checks.

3. [ ] Fix Finding 14 for `0.1.1`.
   1. [x] Identify the browser and input hints available for suppressing mobile text assistance in terminal entry.
   2. [ ] Implement the best supported mitigations without breaking normal terminal input behavior.
   3. [ ] Verify the fix against the local release checks plus real mobile checks.

4. [ ] Refresh the release surface and ship the `0.1.1` patch release.
   1. [ ] Update the release notes, package metadata, artifact references, and any versioned site or README references needed for `0.1.1`.
   2. [ ] Update any package-root docs, screenshots, or public copy that stop being honest once the `0.1.1` fixes land.
   3. [ ] Run the final local verification pass for the patch release.
   4. [ ] Publish `0.1.1` and close the milestone cleanly.

## 2. Verification Checklist

1. [ ] The `0.1.1` release gate is explicit enough to avoid relying on memory, while still fitting the current one-maintainer workflow.
2. [ ] Finding `13` is fixed in a way that actually improves mobile usability instead of just reshuffling cramped controls.
3. [ ] Finding `14` is fixed in a way that materially reduces mobile text-assistance interference during terminal entry.
4. [ ] `0.1.1` ships as a real stabilization release and the public release surface stays aligned with what the patch release actually contains.
