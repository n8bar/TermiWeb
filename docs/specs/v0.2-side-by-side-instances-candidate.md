# 0.2 Side-by-Side Instances Candidate

## Status

- `0.2` candidate
- Not part of `0.1`

## Intent

Allow very wide workstation windows to show more than one live instance at the same time, so a user can watch or work in multiple shared shells without constantly switching the active instance.

## Core Idea

- The primary layout remains one-instance-focused by default.
- On sufficiently wide viewports, the workspace may offer a split-view mode that shows more than one instance side by side, likely starting with one additional visible instance in the first cut.
- Each visible pane is still a real shared instance, not a duplicate local view of the same buffer.
- This candidate does not impose a permanent maximum on how many visible instances the product may support later.

## Constraints

- This is a workstation-first feature, not a phone-first feature.
- It should not degrade the current single-instance experience on narrower viewports.
- It should not turn the UI into a general-purpose tiling window manager.
- Split view should remain compatible with the existing instance model, per-instance width, and shared-session semantics.

## UX Direction

- The feature should appear only when the viewport is wide enough to make multiple panes genuinely usable.
- The default single-instance view should remain the normal path.
- Additional panes should be explicit and reversible, not something that appears unexpectedly.
- Each pane should make it obvious which instance it is showing.
- The design should preserve clear focus and input ownership so typing goes to the intended pane.

## Open Questions

- Should split view be manual only, or should the UI suggest it on very wide windows?
- Should each pane have its own local active-instance selection, or should one pane be primary and the other secondary?
- How should mobile controls and other pane-specific actions behave if only one pane is focused?
- Should additional panes be read-only at first, or fully interactive from the start?

## Non-Goals

- No commitment yet to arbitrary pane counts or a general-purpose tiled workspace in `0.2`
- Arbitrary drag-and-drop pane layout
- Per-device divergence in the underlying shared session model
- Any `0.1` commitment or release dependency
