# 0.2 Side-by-Side Instances Candidate

## 1. Status

1. `0.2` candidate.
2. Not part of `0.1`.

## 2. Intent

1. Allow very wide workstation windows to show multiple live instances at the same time, so a user can watch or work in more than one shared shell without constantly switching away from what they were doing.

## 3. Core Direction

1. The default layout remains one-instance-focused.
2. On sufficiently wide workstation viewports, the workspace may offer a split view that shows multiple instances side by side.
3. Each visible pane is still a real shared instance, not a duplicate local rendering of the same buffer.
4. This candidate does not impose a permanent maximum on how many visible instances the product may support later.

## 4. Constraints

1. This is a workstation-first feature, not a phone-first feature.
2. It should not degrade the current single-instance experience on narrower viewports.
3. It should not turn the UI into a general-purpose tiling window manager.
4. Split view should remain compatible with the existing instance model, per-instance width, and shared-session semantics.

## 5. UX Direction

1. The feature should appear only when the viewport is wide enough to make multiple panes genuinely usable.
2. The default single-instance view remains the normal path.
3. Additional panes should be explicit and reversible, not something that appears unexpectedly.
4. Each pane should make it obvious which instance it is showing.
5. Visible panes should be fully interactive.
6. One pane should still be the focused pane for keyboard input at a time.

## 6. Open Questions

1. Should split view be manual only, or should the UI suggest it on very wide windows?
2. How should pane focus switching work?
3. How should pane-specific controls behave when only one pane is focused for keyboard input?
4. How should instance selection work per pane?

## 7. Non-Goals

1. Any `0.1` commitment or release dependency.
2. Arbitrary drag-and-drop pane layout.
3. Per-device divergence in the underlying shared session model.
4. A commitment yet to arbitrary pane counts or a general-purpose tiled workspace in `0.2`.
